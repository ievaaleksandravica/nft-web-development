The Ethereum Virtual Machine - EVM

What Is A Virtual Machine

NFTs Under The Hood

  • In order to develop NFT Smart Contracts we compile down to a Virtual Machine which acts as a layer of abstraction
  • Virtual Machines are essentially creating a level of abstraction between the executing code and the executing machine
  • This layer is needed to improve the portability of software, as well as to make sure applications seperated from each other, and seperated from their host.

What Are Smart Contracts - Ethereum & Solidity

Smart Contracts

  • Smart Contracts are written in programming language, called solidity
  • Solidity is a language similar to Javascript and C++
  • There are other languages for writing smart contracts like Vyper and Bamboo
  • Before Solidity was released, other languages like Serpent (deprecated) and Mutan (deprecated) were user

Introduction to Opcodes for the EVM

Smart Contracts

  • Smart contract languages like Solidity cannot be executed by EVM directly
  • Instead, they are compiled to low-level machine instructions (called opcodes
  • If you go to EtherVM you can read up on all the Virtual Machine opcodes

Opcodes

  • Smart Contracts are just like regular accounts, except they run EVM bytecode when receiving a transaction, allowing them to perform calculations and further transactions
  • Each opcode is encoded as one byte, except for the PUSH opcodes, which take a immediate value

Storage

  • Storage is a persistent associative map, with uint256s as keys and uint256s as values
  • All contract fields and mappings are saved in storage
  • Storage fields can be inspected using web3.eth.getStorageAt(addtess, key)

How Contract Storage Works With Ethereum Smart Contracts

High Level vs Low Level Languages

  • Lower-level programming languages often use a stack to pass values to functions
  • The EVM uses a 256-bit register stack from which the most recent 16 items can be accessed or manipulated at once.
  • In total, the stack can only hold 1024 items

Byte codes

  • Because of these limitations, complicated opcodes instead use contract memory to retrieve or pass data
  • Contract storage essentially acts as a public database, from which values can be read externally without having to send a transaction to contract (no fees!)
  • However, writing to storage is very expensive (as much as 6000x) to writing in memory

Allow the EVM to Compute Almost Anything

Smart Contracts

  • Under the hood, EVM uses a set of instructions (called opcodes) to execute specific tasks
  • There are over 100 opcodes. This means the EVM is able to compute (almost) anything, given enough resources
  • Because opcodes are one byte, there can only be a maximum of 256 opcodes

Defining an ERC721 Token

Definition

  • An ERC standard is basically a set of rules which enables your contract play to well with other developers' contracts (or Marketplaces, DApps, other software, etc).
  • It provides a guideline to adhere so that the contracts work in accordance to a set of standards allowing for better communication between contracts.
  • Other programmers can write code that interacts with yours without habing to know your entire codebase

Difference between ERC721 and ERC20

  • ERC721 is the number we will associate with NFTs
  • ERC20 is for cryptocurrencies
  • the difference between those two tokens is that ERC 20 is fungible and ERC721 are the non fungible

  • its important that our smart contracts have all the functions that are required in ERC721

Arrays in Solidity - Pop Push and Length methods

Definition of arrays

  • array is a data structure, which stores a fixed-size sequential collection of elements of the same type
  • an array is used to store a collection of data
  • but it is often more useful to think of an array as a collection of variables of the same type

Agenda

  • how to write an array
  • pop, push and length methods
  • remove elements from array

Code example for creating arrays

  • code example of storing information into variables in file arrays.sol, contract learnArrays
  • uint256[] public myArray; empty array for integers
  • uint256[200] public myArray3 fixed array

Pop / Push / Length

  • push() methods adds one or more elements to the end of an array and returns the new length of the array: myArray.push(number)
  • pop() method removes the last element from an array and returns that value to the caller: myArray.pop()
  • length() length is a string property that is used to determing the length of a string: myArray.length

Remove elements from array using delete

What are abstract contracts in Solidity

Abstract Contract Definition

  • Abstract Contract is one which contains at least one function without any implementation
  • Such a contract is used as a base contract
  • Generally an abstract contract contains both implemented as well as abstract functions
  • Derived contract (one that inherit stuff from base contract) will implement the abstract function and use the existing functions as and when required

Code example

  • code example in file abstractContract.sol, contract X
  • if the function has no implementation, we should mark it as virtual and the contract itself as abstract
